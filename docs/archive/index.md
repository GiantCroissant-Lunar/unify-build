# Archive

This section holds historical or internal-reference material that is still useful for maintainers but should not crowd the primary docs surface.

## Archived Categories

- `inbox/` contains raw planning notes that were worth keeping but are not polished docs.
- `removed/` contains notes about behavior that no longer exists.
- `reviews/` contains code-review and evaluation documents captured during larger schema and tooling changes.
- `progress/` contains milestone tracking notes that are no longer part of the main docs flow.

## When to Use This Section

Reach for the archive when you need implementation history, prior review context, or migration background that does not belong in the user-facing guides and reference docs.
